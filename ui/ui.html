Core benchmark capabilities

Four increasingly difficult competencies

VAKRA is designed to test four increasingly difficult competencies that reflect enterprise reasoning behaviors.

01

API Chaining

Compose multi-step tool chains while carrying intermediate results across dependent calls.

02

Tool Selection

Select the correct API from large candidate sets of 100-200 tools using natural-language tool descriptions and API specifications.

03

Multihop

Compose multi-hop API sequence through query decomposition.

04

MultiTurn with Policy adherence

Combine API outputs with document retrieval and reconcile differences in naming and format. Interpret natural-language constraints governing which tools or sources may be used.

Why VAKRA

Benchmarks the reasoning stack, not just the final answer

Enterprise agents must resolve entities, align schemas, retrieve supporting text, select tools under ambiguity, and follow natural-language policies in the same interaction.

API

Structured APIs

8,000+ executable APIs grounded in real databases and enterprise-style tasks.

DOC

Document repositories

Unstructured corpora that condition tool calls and enable cross-source grounding.

POL

Policy constraints

Query-dependent rules that control source selection and operational compliance.
